# PNG-тайлсет: контекстный шаблон и нарезка

Цель: уйти от процедурного пиксель-арта к рисованным/генерируемым
PNG-тайлам. Принцип — как в Tilesetter: художник/нейронка работает не с
изолированными слотами, а с **непрерывным полотном в контексте**: все
варианты стыков лежат в связке с соседями, углы имеют простую плоскую
масочную разметку, стыки сходятся по построению. Никакого заранее
нарисованного арта в шаблоне нет. Скоуп первой итерации: **террейн**
(реки/дороги/здания — процедурный фолбэк, шаблоны второй очереди).
Разрешение ×4: треугольник 14×16 → **56×64**.

## Как устроен шаблон (version 2, mode:context)

Террейн в игре рендерится по dual-сетке треугольников между центрами
гексов; стык = маска трёх углов; слои накладываются по порядку `T`.
Шаблон — 7 полотен пар «верхний материал на канонической подложке»
в порядке пирога:

```
Пляж на Воде · Луга на Воде · Болото на Лугах · Кустарник на Лугах ·
Лес на Кустарнике · Скалы на Лугах · Горы на Скалах
```

Каждое полотно — рваный демо-остров из СМЕЖНЫХ треугольников (та же
геометрия, что в игре, паттерн детерминирован и содержит все варианты
стыков: обе ориентации × маски 1..6 + полные ×2 + подложка). Разметка —
плоские цвета материалов: перерисовывай полотно целиком как одно
изображение, кромку между материалами рисуй по вкусу ВДОЛЬ границы
разметки (нарезчик вырежет по канонической маске с шумом кромки,
идентичным процедурным тайлам — стыки сойдутся и с ними).

Файлы (генерятся кнопкой «Шаблон тайлсета» в дебаг-панели `~`;
актуальные копии закоммичены в `assets/`):

- `tileset_template.png` — полотна с плоской разметкой (вход генерации);
- `tileset_markup.png` — оверлей: сетка треугольников + метки
  канонических тайлов (`l3` = ориентация l, маска 3; `Fr0/Fr1` — полные
  верхние; `fr0/fl0` — подложка). Наложить для ориентировки, убрать
  перед экспортом;
- `tileset_map.json` — геометрия полотен + раскладка ГОТОВОГО
  tileset.png (`out`), по которой работают нарезчик и импортёр.

## Цикл работы

```
1. «Шаблон тайлсета» → tileset_template.png (+ markup + map).
2. Генерация/рисовка ПОВЕРХ полотен (размер листа не менять!).
3. Кнопка «Нарезать лист» → выбрать перерисованный PNG →
   скачается готовый tileset.png.
4. Положить его в assets/ → node build.mjs («PNG-тайлсет: ВКЛ»).
5. Проверка: кнопка «Тайлсет-превью» (атлас с фактической нарезкой)
   и просто карта. Пустые слоты = процедурный фолбэк, можно сдавать
   лист по кусочку (например, сначала только пара «Луга на Воде»).
```

Roundtrip проверен: нарезка самого шаблона даёт карту в плоских цветах
разметки со сходящимися стыками.

## Внутренности (для следующих сессий)

- `19_atlas.js`: `TS_PAIRS` (пары), `tsBlobPattern` (детерминированный
  поиск демо-острова с полным покрытием стыков), `exportTilesetTemplate`
  (полотна+разметка+map), `sliceTilesetSheet` (вырезка канонических
  треугольников: альфа = барицентрическая маска углов + шум кромки как у
  `paintTriTransition`, масштаб ×4), `importSheetDialog` (файл-диалог).
- Выходной tileset.png собирается в раскладке `tilesetSlots()` — its
  же читает импортёр (`loadTileset`/`tsPlace`): слоты с непустой альфой
  замещают процедурные спрайты, атлас растёт до 512×2048.
- `build.mjs::tilesetInject` инлайнит `assets/tileset.png` +
  `tileset_map.json` (v2: раскладка в `out.slots`) как base64 —
  однофайловость, canvas не «портится» (в отличие от file://).
- Геометрия полотна = геометрия игры: столбик x → px `x*56`, ряд y →
  py `y*64 + (нечётная базовая колонка ? +32 : 0)` (zig odd-q).

## Бэклог

- Пары по требованию: сейчас один шаблон на верхний материал (например,
  кромка ГОР рисуется на фоне СКАЛ и переиспользуется везде — в игре
  горы граничат только со скалами, так что это точно; но кромка ЛУГОВ
  над водой и над болотом — одна и та же).
- Реки: контекстное полотно с руслом (вырезка по процедурной маске
  русла), пока — процедурные.
- Дороги (композит 64 масок из 6 сегментов), здания/юниты/логова/фичи.
- Стили кромки нарезки (крисп/шум/перо) параметром.
