# HANDOFF — состояние проекта и план (обновлено после сессии 2026-07-11)

Файл для передачи контекста между сессиями AI-агентов. Прочитай его вместе с
`CLAUDE.md` (правила работы) и `ARCHITECTURE.md` (карта модулей) — этого
достаточно, чтобы продолжить, не читая всю историю.

## Что это за проект

«Дальняя Марка» — браузерная simulation-игра одним HTML-файлом (владелец
тестирует с телефона Android, открывая `index.html`). Поселение переселенцев
на острове: автономная экономика (job pool + рынок труда), автостроитель,
герои/экспедиции/бои, дань империи, рейды. Код: модули `src/*.js` в общем
scope (НЕ ES-модули), склейка `node build.mjs` → `index.html` + `dist/game.js`.

## Сделано в этой сессии: террейн v2.0 по handoff-доку

Реализованы все 7 задач из `docs/fantasy_marka_terrain_handoff.md` §10,
интеграция сразу в игру (по решению владельца), без промежуточного прототипа.

**Кластеры гор и рельеф (`src/02d_relief.js`)**
- Жадная детерминированная кластеризация MTN-гексов (INV-T5, без rng):
  розетки ★ (гекс + 6 соседей) → тройки ▲ (с lookahead «не клеймить тройку,
  если она отбирает клетку у розетки 6/7» — открытый вопрос handoff §3 решён
  «да») → остаток: цепи (полилиния + «плечи» к якорям), кольцевые цепи
  (`ringChain`), массивы, одиночки. Результат: `S.world.mountains`,
  `S.mtnOwner` (owner-карта клеток).
- Псевдовысоты `S.reliefE`: ранги-источники (пик 5.0 / кольцо 4.0 / узел 3.2 /
  хребет 2.6 / массив 2.4 / одиночка 1.8, `CFG.RELIEF.RANK`) → chamfer-релаксация
  → поле 1-Lipschitz (INV-T2, проверяется в headless: `lipViol=0`). Вода — плоский 0.
- `S.elev` для гидрологии: 0.3·прибрежный градиент + 0.7·E → CDF-перераспределение
  (`y=1−(1−p)^CFG.RELIEF.CDF_POW`): доля высоких поясов — дизайнерский параметр.
- Детекторы (§7): котловины (flood fill равнины, замкнутой горами; «секрет» —
  разведал всё кольцо → лог-событие, `checkValleySecrets` из `onNewDay`) и
  перевалы (не-горный гекс между двумя кластерами ≥3 клеток) → `S.world.valleys`,
  `S.world.passes`.
- Влажность `S.moist` (chamfer от пресной воды: озёра + берега рек,
  `CFG.RELIEF.MOIST_R`) + биомы Уиттекера `S.biome` (`BIO`: луговина/степь/
  болотина/тайга; болото = у воды И в низине CDF-высот). Биомы двигают приоры
  WFC (тайга — лес, степь — сухо, болото — без скал) и фичи (пшеница по биому).
- `wgPeaks`: генератор достраивает 1-2 розетки-«мега-вершины» у концов хребтов
  (пограничный случай «мега-вершина + смежные цепи» из §10.2 воспроизводится).

**Реки v3 (`src/02e_rivers.js`, заменил v2 из 03_state_mapgen)**
- Узлы = центры dual-треугольников, высота узла = среднее `S.elev` углов
  (+крошечный хеш-джиттер для меандров). **Priority-flood** от морских узлов
  вверх: у каждого узла ровно один downhill-выход, локальных минимумов нет —
  каждая река гарантированно достигает моря; озёра дренируются через низший
  седловой узел (реки втекают и вытекают сами, стоки озёр — эмерджентные).
- Истоки: седловые треугольники троек (приоритет) и перевалов, периметр
  подножий; детерминированный отбор с спейсингом (`CFG.RELIEF.RIVERS`).
- Flow-аккумуляция: +1 от каждого истока вдоль down-пути; конфлюэнции
  возникают сами (граф тривалентен); ширина тайла w1–3 от потока
  (`wideAt`/`hugeAt`); entity `S.world.rivers` с parent/child при слиянии.
- Контракты сохранены: `S.riverEdges` (Set рёбер-блокеров), `S.riverTris`
  (+поле `w`), `S.riverStats` (+`flowMax`); пафайндинг/мосты/автостроитель
  не менялись.

**Рендер (`19_atlas.js`, `20_render.js`)**
- Гипсометрические пояса (stacked binary passes §5): спрайты `rl{2..5}_{or}_{bits}`,
  один марчинг по порогам E (1.5/2.5/3.5/4.5), материалы: дымка предгорий
  (дизеринг) / склоны M1/M2 / снежник (только розетка пробивает 4.5). Кромка
  пояса L≥3 обведена — читается как crease-линия. Снег из тайла MTN убран.
- **Knowledge-слой (§4):** `buildRelief` считает кластеры и E только по
  `S.explored` горам (`reliefRenderField`) — гора буквально растёт из тумана;
  триггер `S.reliefDirty` из `markVisibleCell`/`exploreRing`, throttle 0.5 c
  в кадре. Симуляция при этом читает истинное `S.reliefE`.
- Реки: спрайты `rt_{tint}_{or}_{mask}_w{1-3}_v{0-1}` — ширина от потока +
  noise-вариант меандра (noisy edges §8.4: изгиб внутри тайла, концы пришиты
  к серединам рёбер — стыки всегда сходятся; вариант — хеш позиции, INV-T4).
- Биом-тинты степи/болотины: дизеринг-оверлеи `bio{2|3}_{or}_{bits}` поверх
  лугов (order 2.5), статичный слой.
- Дебаг-оверлей сеток (~): золотые рёбра гекс-графа, пересекающие русло, —
  кандидаты мостов. Инспектор/тултип показывают биом и снежный пик.
- Атлас вырос до ~546px из 1024 (guard на переполнение в `buildAtlas`).

**Headless-диагностика (`24_headless.js`)**
- Строки `relief:` (кластеры по видам, Emax, lipViol, долины/перевалы),
  `biomes:`, расширенная `rivers:` (flowMax, w1/w2/w3).

## Как проверялось

- `node build.mjs && node dist/game.js test-1 && node dist/game.js test-1 quest`
  + сиды `world-7`, `rnd-42` — без исключений; экономика 60 дней живёт.
- Отдельные проверки: детерминизм кластеров (два `newGame` того же сида →
  идентичная сигнатура), «секрет» долины (форс-разведка кольца → лог),
  knowledge-кластеры (3 из 9 клеток цепи → кластер chain/3).
- Браузер: Playwright + локальный `three.min.js` (CDN недоступен из песочницы;
  скрипты `shot.mjs`/`shot2.mjs` в скретчпаде сессии — паттерн:
  `route('**/three.min.js')` → локальный файл, `page.evaluate(restart/tick)`,
  скриншоты). Скриншоты: розетка с поясами и снежником, остров целиком —
  выглядит по-оскаровски.

## Известные хвосты / не отполировано

- Спрайты растянуты ×2 — ждут перерисовки под новый масштаб (владелец в курсе).
- w3-реки (поток ≥ `hugeAt`=4) на текущих картах редки: сливаются 2-3 истока
  (flowMax≈3). Если хочется больших рек — поднять число истоков или снизить пороги.
- Перевалы пока только entity (`S.world.passes`) + истоки рек: узкое место для
  пафайндинга/обороны и имена — в бэклоге.
- Кольцевые цепи (`ringChain`) детектятся, но генератор их почти не создаёт.
- Биом-тинты — дизеринг поверх лугов; тайга визуально не выделена (лес и так
  гуще у воды). Уиттекер-таблица упрощена до 4 биомов.
- В бэклоге WFC: жёсткие правила соседства (каркас трейса уже пишется).
- `19_atlas.js`/`20_render.js` крупные — пилить по подсекциям при росте.

## План следующих шагов (от владельца, прежний + новое)

1. Перерисовка спрайтов под крупный гекс (тайлы 28×32, здания, деревья).
2. Боевая система: оружие и экипировка героев, эликсиры.
3. Экспансия через игрока: ключевые здания ставит игрок (механизм фазы выбора есть).
4. Развитие entity мира: имена хребтов/рек/долин/перевалов (данные уже есть:
   `S.world.mountains/valleys/passes/rivers`), квесты на их основе.
5. Террейн: декор-слой (§8 хвост — «пушистые» импосторы деревьев/облаков),
   перевалы как игровое узкое место, дороги вдоль контурных линий.
