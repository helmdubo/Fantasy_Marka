# HANDOFF — состояние проекта и план (обновлено после сессии 2026-07-11)

Файл для передачи контекста между сессиями AI-агентов. Прочитай его вместе с
`CLAUDE.md` (правила работы) и `ARCHITECTURE.md` (карта модулей) — этого
достаточно, чтобы продолжить, не читая всю историю.

## Что это за проект

«Дальняя Марка» — браузерная simulation-игра одним HTML-файлом (владелец
тестирует с телефона Android, открывая `index.html`). Поселение переселенцев
на острове: автономная экономика (job pool + рынок труда), автостроитель,
герои/экспедиции/бои, дань империи, рейды. Код: модули `src/*.js` в общем
scope (НЕ ES-модули), склейка `node build.mjs` → `index.html` + `dist/game.js`.

## Сделано в этой сессии (2026-07-11): террейн v2.0 + слияние перерисовки спрайтов

В эту ветку влита параллельная ветка перерисовки спрайтов
(`claude/ascii-art-sprite-redraw-wek6u6`); ниже обе работы.

### Часть 1: перерисовка спрайтов под крупный гекс (влитая ветка)

- Тайлы террейна 14×16 → **28×32** (`TRIW/TRIH` в `19_atlas.js`): поверх шума
  `terrPix` добавлен структурный декор `decorTri` — ёлки двух размеров в лесу,
  пучки трав и редкие цветы на лугах, валуны и трещины на скалах, диагональные
  гребни и снежники на горах, гребешки волн на воде. Снежник рисуется только
  если целиком помещается в маску тайла (иначе белые обрывки).
- Здания/логовища/фичи 16×16 → **32×32** (ратуша и Башня знаний 32×34, корабль
  20×20): все процедурные painter-ы (`paintFisher`, `paintLumber`, …) заменены
  детальными ASCII-гридами в `18_sprites.js` (крыши-шинглы, кладка, светящиеся
  окна, вывески). Гриды сгенерированы скриптом-генератором (скретчпад
  `genart.mjs`), но правятся руками как обычный ASCII-арт.
- Дороги/реки/мост/устье/водопад перерисованы под 28×32: русло ~5px с изгибом,
  дорога 6px с тёмной кромкой и камешками, водопад — струи с рваным краем и
  пенным подножием. Мост теперь ОРИЕНТИРОВАН по дорожной топологии: три
  спрайта `bridge_v`/`bridge_ne`/`bridge_se` (настил вдоль оси дороги N-S или
  диагонали, доски поперёк хода); выбор по паре гексов ребра в `buildRoads`
  (раньше был один горизонтальный настил, не совпадавший с дорогой).
- Юниты и враги экрана боя НЕ трогались (16×16): на карте рисуются в полгекса
  (экранная плотность уже совпадает), в бою — холст 16×16 (`22b_battle_ui`).
- `validateSprites` — таблица размеров по каждому гриду; `bobFrame` не зависит
  от ширины; в `20_render.js` базовый пиксель зданий 16 → 32 (`spr.h/32`).
- Проверка: Playwright-скриншоты атласа и живой карты (день/ночь/старт с
  палаткой/горы/водопад) — скрипты `viewer.html`, `game_shot.mjs`,
  `scenic_shot.mjs`, `atlas_dump.mjs` в скретчпаде сессии.

**Согласование при слиянии с террейном v2.0:**
- `paintRiverTri` объединён: русло 28×32 в их стиле + мои ширина `w` (1..3 →
  ~5/7/9px) и noise-вариант `v`; амплитуда меандра падает с шириной.
- Пояса рельефа L3/L4 получили диагональные гребни в стиле `decorTri`
  (тон по поясу), чтобы гипсометрия не выглядела плоской заливкой поверх
  нового горного арта; снег гор — только пояс E≥4.5 и decorTri-снежники
  на нижних склонах/переходах.
- Атлас 512×**2048** (56 рельефных + 28 биомных + 336 речных тайлов 28×32
  в 1024 не влезали; занято ~1092px); guard на переполнение остался.
  `S.atlasMs` ≈ 385мс на десктопе (на телефоне ждать больше) — только на
  буте/рестарте; если станет мешать, кандидаты на срезание: v-варианты
  рек для w2/w3.

### Часть 1b: PixelLab — сгенерированные Wang-тайлсеты (референсы)

Через PixelLab MCP (`https://api.pixellab.ai/mcp`, ключ у владельца; в конфиг
добавлен `claude mcp add pixellab …`) сгенерирована связная цепочка Wang-тайлсетов
32px, 16 тайлов каждый: вода→луга → {лес, скалы, болото, степь}, скалы→горы —
через base tile ID травы/скал, поэтому все переходы в одном стиле. Лежат в
`assets/pixellab/*.png` (+ `*_metadata.json` с corner-раскладкой), превью —
`docs/pixellab_tilesets_preview.png`. Это КВАДРАТНЫЕ corner-Wang тайлы — в
текущий треугольный dual-grid марчинг напрямую не вставляются; использовать как
арт-референсы для terrPix/decorTri или для будущего квадратного рендера.
Расход: 16 генераций из 2000 по подписке. Заготовка клиента — `plab.mjs` в
скретчпаде (JSON-RPC поверх HTTP, ответы SSE); в живую сессию MCP-сервер
подхватывается только после рестарта, потому ходили curl-ом.

**Интеграция в игру (юниты карты — PNG вместо ASCII):**
- `build.mjs` → `genUnitsPng()` вшивает base64 PNG в автогенерируемый
  `src/18c_units_png.js` (`UNIT_PNG`, ~880KB; index.html ≈ 1.25MB).
- `19_atlas.js`: `loadUnitImages()` декодирует все PNG в Image ДО `boot()`
  (см. `25_boot.js`); `buildAtlas` рисует их в атлас (1024×2560, занято
  ~2310px) под ключами `up_<race>_{idle|walk|work}_<slot>[_frame]`.
- `20_render.js`: `unitHexSlot(dx,dy)` — сектор из 6 гекс-направлений по
  вектору движения; `unitSprPick` — walk-цикл (8к) в движении, work-цикл (9к)
  на работе, иначе idle; поселенцы/рейдеры/партия героев. ASCII-гриды
  остались фолбэком (фейл декода).
- **Кроп холстов (v2.2)**: PixelLab отдаёт холст на 25-40% больше арта
  (запас под размах анимаций; холсты разные: 56/60/64px). При загрузке
  `computeUnitCrops` считает bbox по альфе — ЕДИНЫЙ на группу «раса_анимация»
  (юнион по кадрам/направлениям, иначе анимация дёргается) — и в атлас идёт
  только bbox (юниты: ~1250px атласа → ~250px). Квад юнита строится от
  пикселя арта (0.95wu/56px — одна плотность у всех холстов, это же чинит
  разнобой размеров гном/рейдер), якорь — низ bbox (ноги) и центр холста.
  Метаданные кропа лежат на SPR-записи (cx/cy/cw/chh/canW/canH).
- Экран боя (22b) пока на старой рисовке; боевые анимации (attack/hurt/
  death SE/SW) в ассетах готовы — интеграция боя отдельным шагом.

**Анимации** (`assets/pixellab/characters/<имя>/animations/`):
- `walk`, `work` — 6 гекс-направлений (карта); walk — шаблон `walking-8-frames`
  (8 кадров), work/attack — v3-кастом (9 кадров: референс + 8).
- `attack`, `hurt`, `death` — ОДНА сторона: расы `south-east` (экран боя,
  герои смотрят вправо), raider `south-west` (враг, смотрит влево).
- `death` — v3-кастом «low fantasy»: рубящий удар, брызги крови, оседание,
  финальные кадры — тело в луже крови (11 кадров; последний кадр = idle-кадр
  трупа). Шаблонное `falling-back-death` осталось только на сервере PixelLab.
- Превью: `docs/pixellab_animations_preview.png`.

Заметки: лимит Tier 1 — 8 фоновых задач (v3-персонаж = 2 задачи, анимация =
1 задача на направление). НЕ ставить в очередь пачку animate-вызовов залпом:
сервер молча роняет лишние — сериализовать (очередь → дождаться → следующий).
Листинг `get_character` ОБРЕЗАЕТ список анимаций — источник правды только
`download`-бандл (zip). v3-анимации именуются по animation_name, шаблонные —
по имени шаблона (`animating`/`falling_backward`/`taking_a_punch` в zip).
Прямые backblaze-ссылки ротаций отдают 401 при Authorization-заголовке —
качать `download`-бандл через api.pixellab.ai (zip с rotations/).
Wang-тайлсеты из части 1b — только референсы (квадратные, в треугольный
марчинг не встают); персонажи же ортогональны рендеру террейна и пригодны
для интеграции напрямую (сейчас юниты 16×16 в `18_sprites`, замена на PNG
потребует загрузки внешних ассетов в атлас или конвертации в гриды).

### Часть 2: террейн v2.0 по handoff-доку

Реализованы все 7 задач из `docs/fantasy_marka_terrain_handoff.md` §10,
интеграция сразу в игру (по решению владельца), без промежуточного прототипа.

**Кластеры гор и рельеф (`src/02d_relief.js`)**
- Жадная детерминированная кластеризация MTN-гексов (INV-T5, без rng):
  розетки ★ (гекс + 6 соседей) → тройки ▲ (с lookahead «не клеймить тройку,
  если она отбирает клетку у розетки 6/7» — открытый вопрос handoff §3 решён
  «да») → остаток: цепи (полилиния + «плечи» к якорям), кольцевые цепи
  (`ringChain`), массивы, одиночки. Результат: `S.world.mountains`,
  `S.mtnOwner` (owner-карта клеток).
- Псевдовысоты `S.reliefE`: ранги-источники (пик 5.0 / кольцо 4.0 / узел 3.2 /
  хребет 2.6 / массив 2.4 / одиночка 1.8, `CFG.RELIEF.RANK`) → chamfer-релаксация
  → поле 1-Lipschitz (INV-T2, проверяется в headless: `lipViol=0`). Вода — плоский 0.
- `S.elev` для гидрологии: 0.3·прибрежный градиент + 0.7·E → CDF-перераспределение
  (`y=1−(1−p)^CFG.RELIEF.CDF_POW`): доля высоких поясов — дизайнерский параметр.
- Детекторы (§7): котловины (flood fill равнины, замкнутой горами; «секрет» —
  разведал всё кольцо → лог-событие, `checkValleySecrets` из `onNewDay`) и
  перевалы (не-горный гекс между двумя кластерами ≥3 клеток) → `S.world.valleys`,
  `S.world.passes`.
- Влажность `S.moist` (chamfer от пресной воды: озёра + берега рек,
  `CFG.RELIEF.MOIST_R`) + биомы Уиттекера `S.biome` (`BIO`: луговина/степь/
  болотина/тайга; болото = у воды И в низине CDF-высот). Биомы двигают приоры
  WFC (тайга — лес, степь — сухо, болото — без скал) и фичи (пшеница по биому).
- `wgPeaks`: генератор достраивает 1-2 розетки-«мега-вершины» у концов хребтов
  (пограничный случай «мега-вершина + смежные цепи» из §10.2 воспроизводится).

**Реки v3 (`src/02e_rivers.js`, заменил v2 из 03_state_mapgen)**
- Узлы = центры dual-треугольников, высота узла = среднее `S.elev` углов
  (+крошечный хеш-джиттер для меандров). **Priority-flood** от морских узлов
  вверх: у каждого узла ровно один downhill-выход, локальных минимумов нет —
  каждая река гарантированно достигает моря; озёра дренируются через низший
  седловой узел (реки втекают и вытекают сами, стоки озёр — эмерджентные).
- Истоки: седловые треугольники троек (приоритет) и перевалов, периметр
  подножий; детерминированный отбор с спейсингом (`CFG.RELIEF.RIVERS`).
- Flow-аккумуляция: +1 от каждого истока вдоль down-пути; конфлюэнции
  возникают сами (граф тривалентен); ширина тайла w1–3 от потока
  (`wideAt`/`hugeAt`); entity `S.world.rivers` с parent/child при слиянии.
- Контракты сохранены: `S.riverEdges` (Set рёбер-блокеров), `S.riverTris`
  (+поле `w`), `S.riverStats` (+`flowMax`); пафайндинг/мосты/автостроитель
  не менялись.

**Рендер (`19_atlas.js`, `20_render.js`)**
- Гипсометрические пояса (stacked binary passes §5): спрайты `rl{2..5}_{or}_{bits}`,
  один марчинг по порогам E (1.5/2.5/3.5/4.5), материалы: дымка предгорий
  (дизеринг) / склоны M1/M2 / снежник (только розетка пробивает 4.5). Кромка
  пояса L≥3 обведена — читается как crease-линия. Снег из тайла MTN убран.
- **Knowledge-слой (§4):** `buildRelief` считает кластеры и E только по
  `S.explored` горам (`reliefRenderField`) — гора буквально растёт из тумана;
  триггер `S.reliefDirty` из `markVisibleCell`/`exploreRing`, throttle 0.5 c
  в кадре. Симуляция при этом читает истинное `S.reliefE`.
- Реки: спрайты `rt_{tint}_{or}_{mask}_w{1-3}_v{0-1}` — ширина от потока +
  noise-вариант меандра (noisy edges §8.4: изгиб внутри тайла, концы пришиты
  к серединам рёбер — стыки всегда сходятся; вариант — хеш позиции, INV-T4).
- Биом-тинты степи/болотины: дизеринг-оверлеи `bio{2|3}_{or}_{bits}` поверх
  лугов (order 2.5), статичный слой.
- Дебаг-оверлей сеток (~): золотые рёбра гекс-графа, пересекающие русло, —
  кандидаты мостов. Инспектор/тултип показывают биом и снежный пик.
- Атлас вырос до ~546px из 1024 (guard на переполнение в `buildAtlas`).

**Headless-диагностика (`24_headless.js`)**
- Строки `relief:` (кластеры по видам, Emax, lipViol, долины/перевалы),
  `biomes:`, расширенная `rivers:` (flowMax, w1/w2/w3).

### Часть 3: геймплей-пак v2.2 (правки владельца)

- **Тролль ×1.5** на карте (`UNIT_SCALE` в 20_render; квад масштабируется от
  базовой линии ног).
- **Данжены**: древние руины (F.RUINS) больше не работа поселенца — при
  генерации становятся пассивными логовами `id:'ruins'` (не копят агро, не
  рейдят, не пугают; спрайт — алиас `l_ruins`→`f_6`); герои штурмуют их как
  обычное логово (2 этапа, скелеты). Заброшенные/сгоревшие здания ЛЮБОГО типа
  зачищаются партией: `sendDelve` обобщён (шахта — этажи как раньше; прочее —
  режим `ruinsweep`: один бой с мародёрами, разовая добыча, флаг `b.dgnDone`),
  кнопка «Зачистить развалины» в инспекторе.
- **Шахты у самоцветов**: `resScore` жилы теперь ×8 и только РАЗВЕДАННЫЕ;
  отдельный проход кандидатов вокруг explored-жил + «рудная концессия»
  (siteOk с veinReach: до 5 клеток за зону влияния).
- **Корабль**: курс отплытия/прибытия — 16 лучей из порта, берётся направление
  с самой длинной полосой морской воды (раньше — центроид морской компоненты,
  который у окружающего остров моря лежит ПОСРЕДИ СУШИ: корабль плыл вглубь
  берега).
- **Бой на кулдаунах** (22b_battle_ui переписан): у каждого бойца заряд
  (cd/cdMax: герои 2.3–3.4с по классу, враги 3.1/4.0±0.3). Враги бьют сами
  по заполнении (полоска видна), герой с полным зарядом подсвечен жёлтым и
  атакует ПО КЛИКУ. Засада — враги стартуют почти заряженными («Дозор» съедает
  фору), «Вор» даёт героям +50% стартового заряда и темп первого удара.
  Отход: hp<30%, таймаут 90с или «Отступить». Headless/autoQuest крутит
  старый мгновенный stepBattleRound — ядро общее. Дальше — эволюция в
  пошаговый режим с решениями игрока.

## Крупные правила мира

- Масштаб: карта 64×64, гексы визуально ×2 (`ZOOMS [4,6,8]`), юниты рисуются
  в полгекса (экранный размер прежний). Спрайты перерисованы под новый масштаб:
  тайлы 28×32, здания 32px (см. секцию сессии 2026-07-11).
- Лес непроходим для ЖИТЕЛЕЙ (`S.pass`), герои/партии/рейдеры ходят
  (`S.passHero`, `withHeroPass`). Пеньки проходимы; лес регенерирует из
  пеньков (15-23 дня); лесоруб рубит с опушки (adj).
- Лесопилки: прогрессивная стоимость (`costOf`): 0 действующих → бесплатно
  (фикс дедлока дерева); заброшенные не оживают (энкаунтеры); фермы/причалы
  оживают при восстановлении угодий.
- Старт: палатка переселенцев → фаза `S.phase='scout'` (разведка радиус 6,
  экономика спит) → игрок ставит ратушу тапом (валидация `thSiteScore`,
  кнопка «Авто»; headless — авто через 2 дня) → палатка сворачивается →
  `'play'`. «Губернаторские указы» ensureStarter* удалены.

## Как проверялось

- `node build.mjs && node dist/game.js test-1 && node dist/game.js test-1 quest`
  + сиды `world-7`, `rnd-42` — без исключений; экономика 60 дней живёт.
- Отдельные проверки: детерминизм кластеров (два `newGame` того же сида →
  идентичная сигнатура), «секрет» долины (форс-разведка кольца → лог),
  knowledge-кластеры (3 из 9 клеток цепи → кластер chain/3).
- Браузер: Playwright + локальный `three.min.js` (CDN недоступен из песочницы;
  скрипты `shot.mjs`/`shot2.mjs` в скретчпаде сессии — паттерн:
  `route('**/three.min.js')` → локальный файл, `page.evaluate(restart/tick)`,
  скриншоты). Скриншоты: розетка с поясами и снежником, остров целиком —
  выглядит по-оскаровски.

## Известные хвосты / не отполировано

- Баланс нового мира не вылизан: плотность лесов/скал, темп экономики при
  непроходимом лесе, дальность шахт у гор — крутить по фидбеку с телефона.
- Тап в фазе выбора ратуши ставит её сразу (без подтверждения) — возможны
  мисклики на телефоне; кандидат: двухтаповое подтверждение.
- w3-реки (поток ≥ `hugeAt`=4) на текущих картах редки: сливаются 2-3 истока
  (flowMax≈3). Если хочется больших рек — поднять число истоков или снизить пороги.
- Перевалы пока только entity (`S.world.passes`) + истоки рек: узкое место для
  пафайндинга/обороны и имена — в бэклоге.
- Кольцевые цепи (`ringChain`) детектятся, но генератор их почти не создаёт.
- Биом-тинты — дизеринг поверх лугов; тайга визуально не выделена (лес и так
  гуще у воды). Уиттекер-таблица упрощена до 4 биомов.
- В бэклоге WFC: жёсткие правила соседства (появится настоящий бэктрекинг —
  каркас трейса уже пишется), больше типов тайлов.
- `19_atlas.js`/`20_render.js` крупные — пилить по подсекциям при росте.
