# HANDOFF — состояние проекта и план (обновлено после сессии 2026-07-05)

Файл для передачи контекста между сессиями AI-агентов. Прочитай его вместе с
`CLAUDE.md` (правила работы) и `ARCHITECTURE.md` (карта модулей) — этого
достаточно, чтобы продолжить, не читая всю историю.

## Что это за проект

«Дальняя Марка» — браузерная simulation-игра одним HTML-файлом (владелец
тестирует с телефона Android, открывая `index.html`). Поселение переселенцев
на окраине империи: автономная экономика (job pool + рынок труда),
автостроитель, герои/экспедиции/бои, дань империи, рейды. Код: модули
`src/*.js` в общем scope (НЕ ES-модули), склейка `node build.mjs` →
`index.html` + `dist/game.js`.

## ⚠ Текущий режим: ГЕНЕРАТОР ПРЕВЫШЕ ГЕЙМПЛЕЯ

По решению владельца проект сейчас сосредоточен на новом генераторе карт;
геймплей отложен. Кнопка «Запуск» показывает **пошаговый просмотр
генерации** (genViewer в `22_ui.js`): патчи → роли → высоты → террейн →
реки; красные вспышки = откаты бэктрекинга. Неудачная попытка
останавливается с причиной в лог и красным баннером — владелец жмёт
«Запуск» снова и даёт обратную связь по увиденным шагам. Клик по канве —
закрыть просмотрщик. В headless-режиме генератор сам ретраит до 8 попыток.

## Сделано в этой сессии (WFC-генератор v2)

Дизайн согласован и описан в `docs/wfc-design.md` (research: Gumin, Boris
the Brave/DeBroglie, Merrell, Stålberg, Loren Schmidt; примеры владельца —
`docs/Examples_procgen`). Реализовано:

- **`src/02b_wfc.js` — сетко-независимый WFC-солвер**: домены-битмаски
  (≤16 тайлов), выбор клетки с минимальной энтропией Шеннона (мин-куча с
  ленивой инвалидацией), AC-4-пропагация со счётчиками enabler'ов,
  индексированными СЛОТАМИ соседей (работает и на гекс-сетке, и на
  произвольном графе — обратные рёбра ищутся автоматически), undo-лог,
  бэктрекинг с лимитом, `pins`/`domains`/`onCollapse`, трейс для реплея.
  ВАЖНО: enabler-счётчики по слотам, а НЕ по направлениям — у графа
  патчей все рёбра имеют один dirTag (это был баг первой версии).
- **`src/02c_worldgen.js` переписан** (островной генератор удалён):
  1. Вороной-патчи: Poisson-диск ~30-44 сайтов (`WFC.minDist=8`),
     разбиение multi-source BFS (`S.patchOf`), граф смежности патчей.
  2. WFC №1 — роли патчей на графе: SEA/LOW/WET/UP/HIGH/LAKE; континент:
     случайный морской азимут, краевые патчи с морской стороны запинены
     SEA, остальная суша «уходит за карту»; HIGH только через UP,
     LAKE не у моря; 1-2 пина HIGH вглубь материка (сам HIGH не выживает —
     ему нужны ВСЕ соседи UP/HIGH); валидация слоя: доля моря 10-45%,
     связность суши, наличие нагорья.
  3. WFC №2 — высоты 0..6 (`S.elevL`): единственное правило |Δh|≤1,
     веса гауссом вокруг сглаженного целевого поля ролей + fbm; море
     пин 0, озёра пин 1. Хребты/долины эмерджентны (слой «Loren Schmidt»).
     `S.elev` (float, для рек) = уровень/6 + шум.
  4. WFC №3 — террейн: 9 тайлов (внутренний MTN_NODE пишется как MTN,
     клетки-узлы в `S.world.ridgeNodes`); домены сужены высотой (вода=0,
     MTN только h≥5, ROCK h≥3, SAND только у кромки моря, SWAMP h≤1);
     жёсткая матрица: ЛЕС ГРАНИЧИТ ТОЛЬКО С ЛЕСОМ/ОПУШКОЙ (коридоры для
     жителей by construction — релаксация больше не нужна), ГОРЫ только
     через СКАЛЫ, лес/горы не касаются воды; кастом-ограничение «тонкие
     хребты» через onCollapse (у MTN ≤2 соседей-пиков, узел — до 4).
  5. Реки v2 — без изменений (рёберные, по `S.elev`).
  6. `wgValidate`: доля суши 50-92%, море есть, хребет/лес/скалы в
     границах, связность ЖИТЕЛЬСКОЙ проходимости ≥75% (лес непроходим).
- **Новые тайлы** `T={WATER:0,SAND:1,SWAMP:2,GRASS:3,SCRUB:4,FOREST:5,
  ROCK:6,MTN:7}` — ПОРЯДОК = порядок слоёв рендера (`S.terr[i]>=t` в
  `20_render.js`), менять нельзя. SAND/SWAMP: проходимы, не застраиваются;
  SCRUB (опушка): проходим, застраиваем, скорость 0.85 (эльф 1.0);
  SWAMP скорость 0.6 (тролль 1.0). Рендер: слои
  [SAND2,SWAMP3,GRASS4,SCRUB5,FOREST6*,ROCK7,MTN8], реки 8.5, дороги 9
  (*лес — отдельный меш, единственный мутируемый слой). Спрайты новых
  тайлов — процедурные перекраски в `18_sprites.js` (`terrPix`).
- Один прогон генерации ≈ мгновенный (64×64); ~85% попыток валидны с
  первого раза (типичные фейлы: «коридоры перекрыты», ретрай решает).
- Проверено: `node dist/game.js test-1|test-1 quest|world-7|rnd-42` —
  экономика 60 дней живёт на новых картах; браузер — Playwright-скрины
  (просмотрщик, готовая карта, баннеры успеха/фейла).

## Известные хвосты / не отполировано

- Баланс генератора крутить ПО ФИДБЕКУ владельца с реплеев: доли тайлов
  сейчас ~ вода 26%, луга 25%, кустарник 16%, лес 14%, скалы 13%, горы 3%.
  Лес местами «крапинками» (одиночные клетки в кольце опушки — легально,
  но некрасиво); лечится весами/шумом FOREST или мин-размером кластера.
- Мостов на новых картах мало/нет в smoke-прогонах — реки чаще в горах,
  дороги до них не доходят за 60 дней. Посмотреть при балансировке.
- rnd-42: экономика слабая (дерево 0 к 60-му дню) — геймплей отложен,
  но при возврате к нему начать отсюда.
- Спрайты растянуты ×2 и ждут перерисовки; новые тайлы — перекраски.
- В бэклоге дизайна: поймы через инкрементальный «editable WFC»
  (`docs/wfc-design.md` §6), граф POI для логовищ (Лундстен), corner-тайлы.

## План следующих шагов

1. Фидбек владельца по реплеям генерации → крутить веса/матрицу
   (всё в `02c_worldgen.js`: ROLE_W, веса ролей, шумы).
2. Возможные правки структуры по фидбеку (размер патчей, число нагорий,
   глубина моря, кластеры леса).
3. После приёмки генератора — возврат к геймплею (список прошлой сессии:
   спрайты под крупный гекс, экипировка героев, экспансия через игрока,
   имена/квесты entity мира).

## Как проверять (обязательно перед коммитом)

```
node build.mjs && node dist/game.js test-1 && node dist/game.js test-1 quest
```
Плюс сиды world-7, rnd-42. Браузер: Playwright + локальный three.min.js
(CDN из песочницы недоступен): route('**/three.min.js') → локальный файл,
скрипт-образец — в скретчпаде сессии (shot.js): скрины просмотрщика
генерации и карты, проверка `S.genError`.
