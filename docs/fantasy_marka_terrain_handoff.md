# Fantasy_marka — террейн на dual grid: архитектура и план v2.0

> Handoff-документ для Claude (Fable 5). Контекст: игра «Дальняя Марка» — колония/исследование
> на гекс-сетке. Документ фиксирует итог обсуждения террейн-генератора: слои, кластеры,
> рельеф, реки, детекторы паттернов. Рабочий прототип: `dual_grid_relief_v2.html` (работает,
> подтверждено на мобиле). Референсы: Oskar Stålberg (Townscaper / гекс-планета),
> Amit Patel «Polygon Map Generation».

---

## 1. Базовая геометрия: dual grid

- **Гексы** — слой данных. Axial-координаты, flat-top, прямоугольная выкладка
  (`rMin(q) = -floor(q/2)`, ряд `i = r - rMin(q)`). Соседи:
  `DIRS = [(1,0),(1,-1),(0,-1),(-1,0),(-1,1),(0,1)]`.
- **Треугольники** — дуальная сетка: вершины треугольников = центры гексов.
  На каждый гекс `(q,r)` два треугольника: up `{(q,r),(q+1,r),(q,r+1)}`,
  down `{(q+1,r),(q,r+1),(q+1,r+1)}`.
- Соответствие терминологии Амита: центры гексов = его **corners**, центры
  треугольников = углы гексов = узлы его **corner graph**; гекс-граф = center graph.
- **Реки текут по рёбрам гексов**, т.е. через центры треугольников (граф углов гексов).
  **Дороги/мосты — по рёбрам треугольников**, т.е. через центры гексов (гекс-граф).
  Два графа топологически не совпадают → пересечение дороги с рекой = мост, детектится
  тривиально как ребро гекс-графа, пересекающее русло.
- Граф рек **тривалентен**: в узле сходятся максимум 3 русла (2 притока + 1 сток) —
  конфлюэнции геометрически чистые, flow-аккумуляция без разветвлений вниз.

## 2. Трёхслойная архитектура (инварианты)

**Слой 1 — данные (гексы).** Авторитетный. Тип/уровень поверхности на гекс
(земля < предгорье < гора; уровни расширяемы). Всё остальное — производное.

**Слой 2 — маски (треугольники).** Чисто детерминированный марчинг по битности углов.
Ничего не решает, только читает. Непрерывность гарантирована конструктивно: соседние
треугольники разделяют углы.

**Слой 3 — кластеры / супертайлы / декор.** Детекторы паттернов над слоем 1 порождают
производные сущности (entity-контейнеры) и оверлеи.

Инварианты:
- **INV-T1.** Тайл определяется исключительно сигнатурой углов (Wang-контракт).
  Контекст влияет только на выбор варианта интерьера, никогда — на границу.
- **INV-T2.** 1-Lipschitz по уровням: соседние углы отличаются максимум на 1 уровень.
  Рендер многоуровневости = stacked binary passes, один и тот же набор из
  4 канонических масок (пусто / 1 угол / 2 угла / полная; остальное — ротации)
  на каждый переход, меняется только материал.
- **INV-T3.** Супертайл по периметру обязан выглядеть как composite заменяемых тайлов;
  всё уникальное — строго внутри периметра. Промоушен/демоушен = подмена без швов.
- **INV-T4.** Вариативность — варианты с одинаковой сигнатурой, выбор детерминированным
  хешем позиции. Пул контрактов не растёт.
- **INV-T5.** Вся кластеризация детерминирована: жадный порядок по стабильной сортировке,
  tie-break по минимальному ключу. Один и тот же слой данных → один и тот же результат
  (реплеи, headless-валидация по сидам).

## 3. Иерархия кластеров гор

Структурные единицы (выражаются только в терминах гексов, не треугольников):
- **Розетка** — гекс + все 6 соседей одного слоя → **пик** (★). Центр однозначен.
  6 внутренних треугольников полной маски.
- **Тройка** — 3 взаимно смежных гекса → **узел / седловина** (▲). Разделяют ровно
  один треугольник.
- **Цепь** — компонента остатка, являющаяся путём (все степени ≤2, ровно 2 конца) →
  **хребет**; полилиния по центрам. «Плечи»: конец цепи, смежный с занятым кластером,
  тянется к его якорю.
- **Массив** — компонента с ветвлением/циклом. Пока просто помечается.
- **Одиночка** — изолированный горный гекс.

Правила:
- **Эксклюзивность:** гекс принадлежит максимум одному кластеру. Жадный порядок:
  розетки → тройки → остаток. Приоритет по рангу, tie-break по стабильному ключу.
- **Жизненный цикл:** террейн не деградирует; супертайл после формирования фиксирует
  гексы («гора держит форму»). Гистерезис не требуется в v2.0.
- **Детекция событийная:** только при изменении гекса (или его knowledge-статуса)
  проверяется окрестность радиуса 1–2 (dirty-region), не полный проход по карте.

Известный побочный эффект жадности: «плюс» из 4 гор → тройка + одиночка. Открытый
вопрос: нужен ли lookahead «не клеймить тройку, если она блокирует потенциальную
розетку 6/7».

## 4. Механика исследования (knowledge-слой)

- Симуляция хранит истину; **рендер и кластер-детекция читают только knowledge-слой**
  (что известно игроку). Неизвестный сосед марчится как «не гора» (или отдельный
  слой тумана со своими переходными масками).
- Эмерджентный эффект: открыл крайний гекс хребта → рисуется одиночная гора; открыл
  соседей → маски перемарчились, гора «доросла». Розетка материализуется в пик только
  когда известны все 7 гексов — момент «открытия секрета», подчеркнуть анимацией
  перестройки (ретроактивное изменение виденных тайлов — фича, но без акцента
  читается как баг).
- С рельефом (см. §5) гора буквально поднимается из тумана по мере разведки —
  elevation считается по известным гексам.
- Скрытая котловина (см. §7) — «секрет» в чистом виде: пока кольцо не разведано,
  игрок не знает, что внутри пустота.

## 5. Рельеф: горы — это не тайлы (урок Оскара)

У Оскара горы — непрерывная поверхность: поднятые вершины дуальной сетки +
crease-линии вдоль хребтов; материал «скала» — по крутизне склона, не по типу клетки.
Сетка видна на равнине, растворяется на склонах. В Townscaper на рантайме **нет
WFC-поиска**: тайл = детерминированная функция углов + контекстные флаги; «перестройка
соседей» = пере-разрешение изменившихся сигнатур. WFC у него — оффлайн (Bad North).

Перенос в Fantasy_marka:
- **Псевдовысоты.** Ранги-источники: центр розетки 5.0, кольцо 4.0, узел 3.2,
  хребет 2.6, массив 2.4, одиночка 1.8. Поле: `E = max по источникам (ранг − шаги)`,
  chamfer-релаксация (несколько чередующихся проходов `E = max(E, maxNeighbor − 1)`).
  Автоматически 1-Lipschitz → пояса не рвутся. Равнина у гор сама становится предгорьем.
- **Рендер:** гипсометрические пояса = тот же марчинг масок по порогам E
  (0.5, 1.5, …, 4.5). Снежник возникает сам — только центр розетки пробивает 4.5.
- Полилиния хребта = линия излома нормалей (crease) в 3D-версии.
- Кластер остаётся **логической** сущностью-контейнером; визуально он *выращивает*
  рельеф. Классические ассеты-супертайлы гор, возможно, не нужны вовсе.

## 6. Реки и мосты

Текущий прототип (упрощение): реки по центрам гексов, независимые русла. **В v2.0
переделать** по решению из обсуждения:
- Русла — по **рёбрам гексов** (узлы = центры треугольников). Высота узла =
  среднее/минимум трёх углов-гексов; поле остаётся гладким, монотонный спуск сохраняется.
- Истоки — привязаны к элементам дуальной сетки: седловые треугольники между кластерами,
  треугольники по периметру подножий (узлы-тройки — приоритетные кандидаты).
- **Flow-аккумуляция:** у каждого узла один downhill-выход; поток суммируется вниз;
  ширина ~ `sqrt(flow)` (Амит). Конфлюэнции возникают сами; нагромождения цепочек гор
  порождают речные системы вместо дублирующихся линий.
- Река не занимает клетку — она атрибут ребра смежности гексов: барьер/штраф в
  пафайндинге, мост = апгрейд ребра в существующем road-graph. Мост = ребро
  треугольника (гекс-графа), пересекающее русло — подсветка таких рёбер.
- Сток: вниз по градиенту до низины, по низине — детерминированно к краю карты/морю.
  Инвариант Амита на будущее: отсутствие локальных минимумов (или carve) гарантирует,
  что каждая река достигает стока.
- Зигзаг русла с периодом сетки + noisy edges (§7) = органичные меандры.

## 7. Детекторы паттернов и реляционные сущности

Второй механизм «жизни» у Оскара (дворики, бельевые верёвки): рукописные детекторы
поверх базового марчинга. Всё детерминированно, событийно, реплеится по сиду.

- **Котловина/долина:** flood fill равнины, полностью замкнутой горами → промоушен
  внутренних тайлов в «долину». Озёра держать плоскими при расчёте E (Амит) —
  вокруг сами формируются долины, реки втекают/вытекают естественно.
- **Перевал:** два пика/хребта с ровно одним не-горным гексом между ними → проходной
  тайл, узкое место для пафайндинга и обороны.
- **Предгорья:** автоматический пояс у концов хребтов (уже эмерджентно из E).
- **Реляционные сущности** — оверлеи, привязанные к парам/наборам якорей, не тайлы
  (хребет-полилиния уже такая; верёвка между башнями у Оскара — тот же класс).

## 8. Уроки Амита (принято в план v2.0)

1. **Flow-аккумуляция** с конфлюэнциями (§6).
2. **Перераспределение высот по CDF** (сортировка углов + `y = 1−(1−x)²` или иная
   целевая кривая): доля скал/предгорий — дизайнерский параметр, не случайность.
3. **Влажность → биомы:** chamfer-расстояние до пресной воды (те же проходы, что для E);
   2D-таблица Уиттекера (высота × влажность) → тайга/степь/болото без новых механизмов.
4. **Noisy edges:** четырёхугольник «ребро гекса × дуальное ребро треугольника» —
   безопасная зона шума линии; прячет регулярность сетки, не трогая топологию данных.
5. Дороги вдоль контурных линий (перпендикулярно рекам) — эстетика + смысл.
6. Философия: граф несёт геймплейные констрейнты, шум — только несвязанную вариативность
   (совпадает с constraint-first подходом проекта).

Декор-слой (Оскар): деревья и облака — один «пушистый» рендер-примитив (импосторы),
рассыпанный хешем по лесным ячейкам и небу. Четвёртый слой, чистая декорация.

## 9. Состояние прототипа `dual_grid_relief_v2.html`

Реализовано (single-file HTML, canvas, тап-френдли, офлайн):
- Гекс-сетка 13×9, тап переключает гора/равнина; дуальные треугольники, марчинг масок
  (1 угол — уголок, 2 — трапеция, 3 — полная) с автоматической непрерывностью.
- Жадная детерминированная кластеризация: розетки ★ → тройки ▲ → цепи-хребты
  (полилинии с плечами к якорям кластеров) → массивы (пунктир) → одиночки. Статистика.
- Псевдовысоты (ранги → chamfer), гипсометрический рендер stacked binary passes,
  снежники; переключатель «Рельеф вкл/выкл» (наглядно: рельеф — производный слой).
- Реки v1 (по центрам гексов, независимые, ширеют по течению) — **заменить** на §6.
- Смоук-тест: node-harness со стабами DOM, демо + 20 случайных карт без исключений.

## 10. Открытые задачи v2.0 (приоритет сверху вниз)

1. Реки на рёберном графе гексов + flow-аккумуляция + ширина от потока + мосты
   как подсветка пересекающих рёбер треугольников.
2. Пограничные случаи кластеризации: кольцевые цепи (цикл), мега-вершина + смежные
   независимые цепи, нагромождения цепей; правило lookahead тройка-vs-розетка.
3. Детектор котловины (flood fill) + перевала.
4. Влажность + биомная раскраска (Уиттекер).
5. CDF-перераспределение высот.
6. Knowledge-слой в прототипе (туман, прогрессивное раскрытие, промоушен по разведке).
7. Noisy edges.
