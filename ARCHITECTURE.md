# Архитектура «Дальней Марки»

Прототип simulation-игры: пограничная марка на краю империи. Поселенцы сами берут
задачи, автостроитель развивает поселение, внешний мир давит данью, рейдами и логовищами.

## Устройство кодовой базы

Игра собирается из модулей `src/*.js` в **один файл `index.html`** (запуск в браузере,
в т.ч. на телефоне) и `dist/game.js` (headless-прогон в Node).

**Ключевой инвариант:** модули — это НЕ ES-модули. Это обычные скрипты, склеиваемые
в один общий scope в порядке, заданном массивом `MANIFEST` в `build.mjs`. Никаких
`import`/`export` — функции и константы одного модуля напрямую видны из других.
Function declarations хоистятся по всему склеенному файлу, поэтому «поздний» модуль
может вызываться из «раннего» (например, `03_state_mapgen` вызывает `recomputeVision`
из `05_vision`). А вот `const`/`let` доступны только после своего модуля.

```
node build.mjs           # src/*.js + src/shell.html -> index.html, dist/game.js, docs/function-index.md
node dist/game.js seed   # headless smoke-test: 60 игровых дней, сводка экономики
node dist/game.js seed quest  # + сценарий героев: разведка/штурм/обнос логовищ, спуск в шахту
```

## Глобальное состояние

| Глобал | Где объявлен | Что это |
|---|---|---|
| `CFG` | `00_config.js` | Все игровые константы: ресурсы, расы, здания, стоимости, дань, рейды, логовища |
| `T`, `F` | `00_config.js` | Enum-ы террейна (вода/луга/лес/скалы/горы) и фич (ягодник, жила, руины…) |
| `S` | `03_state_mapgen.js` | **Всё состояние игры.** Пересоздаётся `newGame(seed)`. Карта, запасы, поселенцы, здания, логовища, рынок, лог |
| `ATLAS`, `SPR`, `ICONS` | `19_atlas.js` | Canvas-атлас спрайтов и регистр регионов (только браузер) |
| `R` | `20_render.js` | Состояние рендера three.js: сцена, камера, меши, зум (только браузер) |
| `IS_BROWSER` | `00_config.js` | Ветвление браузер/Node; весь рендер и UI за этим флагом |

Детерминизм: вся генерация и симуляция сидированы (`mulberry32`), `S.rng` — единый
источник случайности симуляции. Один сид → одинаковый мир и прогон.

## Поток управления

**Браузер** (`25_boot.js` → `boot()` в `23_main.js`):
`newGame` → `buildAtlas` → `initRender` → цикл `requestAnimationFrame`:
фиксированный шаг `tick(CFG.STEP)` по аккумулятору → dirty-флаги (`S.terrDirty`,
`S.bldDirty`, `S.fogDirty`…) → перестройка мешей → `updateUI`.

**Node** (`25_boot.js` → `runHeadless()` в `24_headless.js`): те же `newGame` + `tick`,
без атласа/рендера/UI.

**Сердце симуляции — `tick(dt)` в `17_daycycle.js`:** часы дня/ночи → `onNewDay`
(дань, торговля, логовища, крафт, иммиграция) → `settlerTick` каждого поселенца →
`warbandTick` рейдеров → `partyTick` экспедиций → `portAutoTick` → `settleThink`
(автостроитель) → `rebuildJobs` + `rebuildMarketFromJobs`.

**Экономическая петля:** здания и стройки публикуют потребности в job pool
(`09_jobs.js`) и на «рынок» офферов (`06/08_market`); поселенец в `settlerTick`
выбирает задачу по полезности (`pickJob`/`marketSelectLaborJob`), идёт через A*
(`findPath` в `02_hex.js`), работает/несёт (`10_hauling.js`), устаёт и отдыхает.

## Карта модулей

Порядок = порядок склейки (см. `MANIFEST` в `build.mjs`).

| Модуль | Ответственность |
|---|---|
| `00_config.js` | `CFG` и справочные таблицы. Баланс правится здесь |

| `02_hex.js` | Гекс-математика (flat-top, odd-q): координаты↔мир, соседи, дистанции, LOS, A* (речные рёбра: мост = дорога на обоих берегах; два слоя проходимости — лес блокирует жителей, но не героев), `colTris` |
| `02c_worldgen.js` | Генератор мира: остров, хребты-«снежинки» (граф-entity), озёра, WFC-заполнение от контуров с приорами от пояса E и биомов; `S.world` (ranges/mountains/rivers/lakes/forests/valleys/passes) |

## Как расширять

**Новое здание:** стоимость/жильё/описание в `CFG` (`00_config`), спрайт-грид в
`18_sprites` + отрисовка в `19_atlas` (`buildAtlas`), правила площадки `siteOk` и
логика автостроителя в `12_settle`, производство/работа в `09_jobs`
(`harvestCycle`/`completeJob`), при необходимости UI-название в `22_ui`.

**Новая механика симуляции:** отдельный модуль `src/NN_name.js`, регистрация в
`MANIFEST` (обычно между `16_party` и `17_daycycle`), вызов из `tick`/`onNewDay`
в `17_daycycle`. Состояние — поля в `S` (инициализация в `newGame`).

**Проверка изменений:** `node build.mjs && node dist/game.js test-1` — прогон
60 дней должен пройти без исключений; сравните сводку до/после. Для геройского
контура добавьте аргумент `quest`. Инварианты гекс-сетки проверяет `hexSelfTest`.
